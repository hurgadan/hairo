## Фаза 2 — Сквозной срез генерации (ядро)

Цель: доказать пайплайн end-to-end на плейсхолдер-промптах, без каталога и визарда.

- [x] `photos`: загрузка селфи → S3-совместимое хранилище — бэкенд (`POST/GET/DELETE /photos`, owner-scoped, presigned URL, MinIO dev/R2 prod) + гостевые сессии (`POST /auth/guest`) + веб-экран загрузки на реальном API
- [x] `generation`: пайплайн по фиксированному референсу (без каталога/визарда, плейсхолдер-фрагмент причёски) — улучшение фото (Промпт B) + рестайлинг (Промпт C) одной цепочкой, без отдельного экрана; при сбое любого из двух шагов — весь job `Failed` (фолбэка на исходное фото нет, фронт показывает ошибку). Очередь — fire-and-forget (как `face-analysis`, `TECH.md`), API `POST /generation` (202) + `GET /generation/:id` (поллинг). Backend-only, ещё нет веб-экрана ожидания/результата.
- [ ] Минимальный UI: загрузил → подождал → увидел результат *(загрузка готова; ожидание/результат — после `generation`)*
- [ ] GDPR-база: согласие, политика, удаление фото *(согласие + удаление фото — есть на API; политика/сроки хранения — TODO)*

**Выход:** «вау-картинка» проходит весь путь автоматически. Это же — демо для B2B-разведки.
