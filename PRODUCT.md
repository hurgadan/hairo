## 2. Визард вопросов-ответов

**Принцип:** что можно — детектим из фото (vision-анализ), спрашиваем только предпочтения. Чем меньше шагов, тем выше конверсия. Целевой объём — автодетект + **5–6 коротких вопросов**.

### 2.1. Автодетект из селфи (не спрашиваем)

Vision-анализ загруженного фото определяет текущее состояние волос/лица → подставляется в фильтрацию (см. `CATALOG.md` раздел 5):

| Детектим | Используется для |
|---|---|
| форма лица | ранжирование `flatters_face_shapes` |
| текущая длина | база для «короче / длиннее» |
| текущая текстура | ранжирование `works_on_textures` |
| текущая густота (грубо) | ранжирование `suits_hair_density` |
| текущий цвет | база для смены цвета |
| предполагаемая гендер-подача | префилл шага 1 |

UX: показать «Мы определили: овальное лицо, средняя длина, волнистые… — поправить?» с возможностью ручной правки.

### 2.2. Вопросы (предпочтения)

| Шаг | Вопрос | Варианты | → влияет на |
|---|---|---|---|
| 1 | Какие образы показать | Женские / Мужские / Все (префилл из детекта) | жёсткий фильтр `gender_presentation` |
| 2 | Желаемая длина | Короче / Как сейчас / Длиннее / Не важно | жёсткий фильтр `length` |
| 3 | Насколько смелая смена | Лёгкое обновление / Заметная / Кардинально | ранжирование (близость к текущему) |
| 4 | Повод / вайб (мультивыбор) | Повседнев / Деловой / Событие / Смелый эксперимент | ранжирование `occasion` + `aesthetic` |
| 5 | Готовность к укладке | Минимум / Средне / Готов(а) заморочиться | жёсткий потолок `maintenance` |
| 6 | Цвет | Оставить свой / Сменить → выбор семейства (`ColorOption`) | отдельный слой цвета (можно отложить до выбора образа) |

**UX-правила:**
- У каждого шага есть дефолт; кнопка «Просто покажи варианты» пропускает опрос (мягкие дефолты).
- Шаг 6 (цвет) можно вынести после выбора образа — цвет не участвует в фильтрации образов.
- Результат: топ-N образов по score (см. ниже), обычно **6–12** карточек, с «показать ещё».

## 3. Логика фильтрации

➡️ Полностью описана в **[CATALOG.md](CATALOG.md) раздел 5**: жёсткие фильтры (gender / length / maintenance) + мягкое ранжирование по score (форма лица, текстура, густота, вайб/повод, популярность). Показываем топ-N, не отсекаем лишнего. Маппинг «ответ визарда → фильтр» — в таблице 2.2 выше.

## 4. Детальный пользовательский флоу

**Принцип:** дать дойти до «вау» с минимумом трения — браузинг и подбор бесплатны, регистрацию и оплату просим как можно позже. Первая генерация — бесплатно (крючок виральности).

### 4.1. Сквозной флоу (экран за экраном)

| # | Экран | Что происходит | Стоимость |
|---|---|---|---|
| 1 | **Вход / лендинг** | Value prop, примеры «до/после», CTA «Примерить причёску» | — |
| 2 | **Загрузка селфи** | Загрузка/съёмка фото, подсказки (анфас, свет), **согласие на обработку фото (GDPR)** | бесплатно |
| 3 | **Улучшение фото** | Авто-улучшение качества, показываем до/после (мини-вау) | бесплатно |
| 4 | **Автодетект** | «Мы определили: овальное лицо, средняя длина…» + правка | бесплатно |
| 5 | **Визард** | 5–6 вопросов предпочтений (раздел 2) | бесплатно |
| 6 | **Подбор (каталог)** | 6–12 карточек-превью по score, лайк/выбор, «показать ещё» | бесплатно |
| 7 | **Выбор образа + цвет** | Выбор одного/нескольких образов + опц. цвет (`ColorOption`) | бесплатно |
| 8 | **Генерация** | Гейт монетизации (см. 4.2). Прогресс/очередь | 1-я бесплатно, далее N кредитов |
| 9 | **Результат** | Сгенерированное фото (вау). Действия: скачать, шарить, ещё образ/цвет, в галерею | — |
| 10 | **Шаринг** | Поделиться с брендингом + реферальная ссылка | — |
| 11 | **Пополнение** | Stripe checkout, пакеты кредитов | оплата |
| 12 | **Аккаунт / галерея** | История генераций, баланс | — |

### 4.2. Точки монетизации

- **M1 — первая генерация бесплатно** (крючок: пользователь видит «вау» до оплаты).
- **M2 — каждая последующая генерация** = списание N кредитов (1 кредит = 1 **примерка** = 1 образ × 1 цвет; доп. вариации — отдельно). Пользовательский термин — **«примерка»**.
- **M3 — недостаточно баланса** → экран пополнения (Stripe), затем возврат к генерации.
- **M4 — реферал** → бонусные примерки пригласившему и новому юзеру (питает M2, см. 4.4).
- Улучшение фото и подбор — всегда бесплатно. (Премиум-апселлы вроде HD-скачивания/пакета вариаций — позже.)

**Цены и бонусы (плейсхолдеры из дизайна, валидировать финмоделью — `PROJECT.md` §8):**
- Пакеты: 1 примерка €2,90 · **5 примерок €8,90** (€1,80/шт, «выгодно») · 15 примерок €19,90 (€1,33/шт).
- Кредиты **не сгорают**.
- Реферал: пригласивший и приглашённый получают **по +2 примерки**; «поделись результатом → +2 примерки».
- В карточке образа score фильтрации показываем как **% совпадения** (напр. 96%).

### 4.3. Когда просим регистрацию

- **Telegram Mini App:** авторизация автоматическая (initData) — трения ноль.
- **Web:** даём пройти шаги 1–7 анонимно; регистрацию (email) просим в момент **сохранения/шаринга результата или первой генерации** — то есть на пике мотивации, когда «вау» уже близко.

### 4.4. Виральная петля

- Результат шарится с **брендингом/вотермаркой** Hairo → каждая картинка = реклама.
- **Реферальная ссылка**: пригласил друга → оба получили бонусные кредиты.
- Возможный буст: «ещё N бесплатных генераций за шаринг результата».

### 4.5. B2B-режим (салон) — кратко

Тот же флоу, но аккаунт и баланс — у салона; клиент в кресле примеряет образы с планшета мастера. Детализируется позже (после B2C-ядра).
