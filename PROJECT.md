# Hairo — AI-примерка причёсок

> Рабочий документ проекта. Описание идеи, цели, целевая аудитория, рынок и первичный бизнес-разбор.
> Дата создания: 2026-06-24.

---

## 1. Идея продукта

Сервис, который по селфи пользователя генерирует его фото с разными причёсками.

**Пользовательский флоу:**
1. Пользователь загружает своё селфи.
2. Система с помощью модели (кандидат — **Nano Banana / Gemini 2.5 Flash Image**) улучшает качество фото. Промпт стандартный для этой задачи.
3. Запускается пошаговый визард вопросов-ответов: предпочтения по причёске, длина волос, толщина волоса, цвет волос (список вопросов прорабатывается отдельно).
4. В системе есть **каталог типов причесок** — его нужно сформировать отдельно. Это отдельная задача для работы с LLM (причёски можно каталогизировать/таксономизировать).
5. На основе ответов система фильтрует каталог и предлагает подходящие типы причёсок.
6. Пользователь выбирает понравившиеся варианты.
7. Система отправляет в контентную модель референс-фото пользователя + референс выбранной причёски + типовой промпт (возможно, дополненный данными из ответов). Модель генерирует фото пользователя с выбранной причёской.

**Монетизация:** генерации платные. Предварительно нужно пополнить счёт (баланс) для генерации.

## 2. Целевая аудитория

- **Парикмахеры / салоны** — могут предложить клиентам доп. услугу (примерка образа перед стрижкой/окрашиванием).
- **Физлица** — самостоятельно генерируют новые образы для себя.

## 3. Рынок

- Старт: **Германия** (где живёт основатель).
- Далее: **Швейцария, Австрия** (DACH).
- Потенциально — другие страны ЕС.

## 4. Бизнес-разбор (первичный)

> Цифры — порядки величин из знаний модели (до начала 2026), не свежий ресёрч. Требуют верификации.

### 4.1. Объём рынка

**B2B (салоны), DACH:**
- Германия: ~80 000 парикмахерских предприятий, ~240 000+ занятых.
- Австрия: ~9 000 салонов.
- Швейцария: ~11 000.
- **Итого DACH: ~100 000 салонов** — реалистичный TAM по B2B.

**B2C (физлица):** на порядки больше (десятки миллионов), но это другая дистрибуция (App Store, TikTok, ASO), а не email.

**Конкуренты (рынок не пустой):**
- **Perfect Corp (YouCam)** — AR-примерка причёсок/цвета для салонов, есть API и B2B-продукт.
- **L'Oréal Style My Hair / ModiFace**.
- Множество мобильных «Hairstyle AI» приложений в B2C.

**Отличие Hairo:** генеративная модель даёт фотореалистичный единичный кадр, а не AR-маску → выше качество. Нужно явное позиционирование против этих игроков.

### 4.2. Юнит-экономика

- Gemini 2.5 Flash Image («Nano Banana»): **~$0.04 за изображение**.
- Сессия = улучшение фото + 3–6 вариантов ≈ **$0.15–0.30 COGS**.
- Цена пакета/подписки €5–15 → **валовая маржа 90%+**.
- Пример: €9 за сессию, COGS €0.25 → маржа ~€8.7.

Себестоимость генерации проблемой не является. **Главный риск — стоимость привлечения (CAC) и дистрибуция.**

Грубый B2B-сценарий: 100 000 салонов × 0.5% проникновения = 500 салонов × €49/мес = **~€25k MRR**. Скромно, но жизнеспособно для соло-основателя.

### 4.3. ⚠️ Юридический риск: email-рассылки по купленным базам

Гипотеза «купить базу адресов и сделать рассылку» в DACH **опасна**:
- **Холодный B2B-email без согласия в Германии практически запрещён** — §7 UWG (исключение узкое — существующие клиенты).
- Покупные базы + рассылка → высокий риск **Abmahnung** (предупреждение от конкурентов / Wettbewerbszentrale, реальные издержки в сотни–тысячи €). В Германии поставлено на поток.
- **GDPR**: обработка купленных персональных адресов требует правового основания и уведомления субъекта об источнике.

**Вывод: массовую email-рассылку по купленным базам снять со стола.**

### 4.4. Легальные каналы продвижения в DACH (без платной рекламы)

1. **Instagram / TikTok органика** — парикмахеры живут в Instagram; контент «до/после» вирусится. Канал №1, почти бесплатный.
2. **Отраслевые выставки**: Top Hair (Дюссельдорф), Hair & Beauty (Франкфурт).
3. **Innungen / Friseurverband** — партнёрства, вебинары.
4. **Дистрибьюторы материалов** (Wella и др.) — доступ к салонам через доверенный канал.
5. **Личный аутрич в DM/LinkedIn** — серая, но менее токсичная зона, чем массовая база.
6. **Freemium-крючок** — бесплатные N генераций, салон сам приводит клиентов.

### 4.5. Итог

- Рынок есть (~100k салонов DACH + крупный B2C), экономика генерации отличная (маржа 90%+).
- Главный риск — **дистрибуция и легальность аутрича**, а не себестоимость.
- Конкуренты (Perfect Corp и др.) уже в B2B — нужно чёткое отличие.

### 4.6. Дистрибуция: ключевой вывод и решение

**Главный стратегический факт:** продукт технически собирается за выходные на Nano Banana → низкий барьер входа → много конкурентов. Защита бизнеса не в продукте, а **в дистрибуции**. Канал привлечения = и есть бизнес.

**Каналов «быстро + предсказуемо + бесплатно + легально» не существует** — любой жертвует чем-то одним. Отсечено: email-рассылки (нелегально, §7 UWG), органика в соцсетях как *основной* (долго/непредсказуемо), выставки (дорого, нет опыта).

**Решение (2026-06-24): wedge = B2C self-serve + параллельная B2B-разведка ногами.**
- **B2C self-serve (основной масштабируемый канал):** продукт напрямую людям через web/app. Виральность встроена в механику, а не «дополнительный маркетинг»:
  - каждая сгенерированная картинка несёт брендинг/вотермарку при шаринге;
  - реферальные бонусы генераций («поделись — получи ещё»);
  - условие работы цикла — реально вау-результат, иначе шаринга нет.
- **B2B прямые визиты (разведка боем):** зайти ногами в 10–20 локальных салонов, показать демо на планшете. Легально, бесплатно, мгновенный фидбэк. За неделю проверяет реальность спроса до того, как написано много кода. Не масштабируется — используется как валидация и источник первых платящих, не как канал роста.
- **Резерв на потом:** интеграция в салонный софт (Treatwell / Shore / Phorest) — один партнёр = сотни салонов, но долгие переговоры.

### 4.7. Качество модели — валидировано

Качество генерации **уже подтверждено вручную** (Валерия, жена основателя, тестировала механизм и модели — результат «супер»). Самый дорогой технический риск снят до начала разработки. Её ручные промпты/референсы = готовый прототип пайплайна и основа каталога причёсок → **сохранить как актив проекта**.

### 4.8. Платформа и Telegram-канал

**Решение: web-first.** У основателя нет мобильного опыта; web покрывает все сценарии и **дёшево оборачивается в Telegram Mini App** без переписывания. Архитектуру держим web-first → опция Telegram открыта почти бесплатно.

**Поправка по Telegram как каналу дистрибуции:**
- Для *коренного* DACH-рынка Telegram слабый — там доминирует **WhatsApp (~90%+)**, Telegram нишевый.
- Где Telegram реально работает — **русско-/украиноязычная диаспора в Германии** (миллионы людей, Telegram-native, много бьюти-мастеров). Тут у основателей **нечестное преимущество**: язык + экспертиза + сеть Валерии.

**Итоговое распределение каналов:**
- **Telegram → диаспора-плацдарм (beachhead):** тёплый старт, где есть преимущество.
- **WhatsApp / web → выход на коренной DACH-рынок** (следующий этап).
- **Web-first архитектура обслуживает оба.**

### 4.9. Плацдарм: пан-европейская русскоязычная диаспора

Beachhead шире, чем Германия. Русскоязычная диаспора **по всей Европе** (Германия, Испания и др.), Telegram-native. Языковое покрытие основателей: **RU / ES / DE** (Валерия знает испанский, в Испании крупная русскоязычная диаспора). Это **пан-европейский диаспора-wedge в Telegram** с языковым преимуществом, которого нет у конкурентов. Защищаемый старт, предпочтительнее «весь DACH сразу».

### 4.10. WhatsApp vs Telegram как продуктовая поверхность

WhatsApp **не может разместить web-app внутри себя** так, как Telegram Mini Apps:
- **Бот:** WhatsApp Business Cloud API есть, но писать первым без opt-in нельзя; вне 24ч окна — только одобренные шаблоны, **платно за беседу**; верификация Meta Business, часто через BSP. Telegram Bot API — бесплатно, свободные сообщения.
- **Mini App:** у WhatsApp **аналога нет**. WhatsApp Flows = ограниченные формы/экраны, не произвольный web-app. Максимум — ссылка во внешний браузер. Telegram Mini Apps = твой web-app в webview, бесплатно.

**Следствие — подтверждает web-first:** обычный отзывчивый web-app = универсальная поверхность. Telegram встраивает его как Mini App, WhatsApp лишь ссылается, коренной DACH открывает в браузере. **Одна кодовая база.** WhatsApp использовать только как канал транзакционных уведомлений/ссылок, не как поверхность приложения.

### 4.11. Биллинг и юр-статус

- У основателя есть **Einzelunternehmen (ИП) в Германии** → юр-лицо для приёма платежей решено.
- **Stripe** — выбранный провайдер: подписки + разовые пополнения баланса генераций нативно.
- Флаг на будущее (не сейчас): цифровые услуги физлицам в ЕС облагаются **НДС по стране клиента** (схема OSS/MOSS); закрывается через Stripe Tax + Steuerberater при росте выручки.
