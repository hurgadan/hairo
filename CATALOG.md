# Hairo — Каталог причёсок и таксономия

> Доменная модель каталога: оси классификации, схема карточки, словари значений, модель «подходящести» и логика фильтрации.
> Продуктовый флоу и визард — в `PRODUCT.md`. Реализация хранения — `ARCHITECTURE.md` (модуль `catalog`).
> Статус: спецификация (Фаза 0). Сам контент каталога генерируется отдельно как задача под LLM.

---

## 0. Решение по моделированию

- **Каталог = именованные образы** (узнаваемые причёски), каждый описан структурными тегами.
- **Цвет волос — отдельный независимый слой** (не плодим один образ в разных цветах).
- Пользователь выбирает **образ** (из отфильтрованного списка) и **цвет** (отдельно). В генерацию идёт: фото пользователя + **безликий** референс образа + фрагмент цвета + промпт-шаблон (см. раздел 3.1).
- Любой образ технически генерируем на любом лице → «подходящесть» это **рекомендация/ранжирование**, а не жёсткий запрет. Жёстко отсекаем только по явным предпочтениям пользователя (нежелательная длина, пол-презентация, цвет).

---

## 1. Оси классификации (структурные теги образа)

Простым языком — что описывает каждая ось:

| Ось | Что это | Допустимые значения (controlled vocabulary) |
|---|---|---|
| **length** | длина волос | `buzz` (очень коротко), `short` (короткие/пикси), `chin` (до подбородка/боб), `shoulder` (до плеч/лоб), `mid` (ключица–грудь), `long` (ниже груди) |
| **silhouette** | общая форма/силуэт | `rounded`, `graduated` (с градуировкой), `blunt` (ровный срез), `tapered` (сходящий), `disconnected`, `a-line`, `shag` (каскад-лохматый), `uniform` |
| **texture** | какую текстуру показывает образ | `straight`, `wavy`, `curly`, `coily` (афро) |
| **fringe** | чёлка | `none`, `curtain` (шторка), `blunt` (ровная), `side-swept` (косая), `wispy` (редкая), `baby` (микро), `full` (густая) |
| **layering** | слои | `one-length` (без слоёв), `layered` (слои), `heavily-layered` (сильная каскадность) |
| **parting** | пробор | `middle`, `side`, `deep-side`, `none` (зачёс/без пробора) |
| **finish** | укладка/подача | `sleek` (гладко), `blowout` (объёмная укладка), `tousled` (небрежно), `slicked-back`, `natural`, `updo` (собранные), `braided` (косы), `wet-look` |
| **volume** | объём/структура | `flat`, `medium`, `voluminous` |
| **gender_presentation** | гендер-подача | `feminine`, `masculine`, `unisex` |
| **aesthetic** | вайб/эстетика (мультитег) | `classic`, `edgy`, `romantic`, `professional`, `trendy`, `retro`, `natural`, `glam`, `minimal` |
| **era** | отсылка к эпохе (опц.) | `90s`, `y2k`, `70s`, `60s`, `modern`, … |
| **occasion** | повод (мультитег) | `everyday`, `work`, `event`, `bold` |
| **maintenance** | сложность ухода/укладки | `low`, `medium`, `high` |

## 2. Оси «подходящести» (для ранжирования, не для жёсткого отсечения)

Хранятся на карточке как списки совместимости:

| Ось | Значения | Смысл |
|---|---|---|
| **flatters_face_shapes** | `oval`, `round`, `square`, `heart`, `oblong`, `diamond` | каким формам лица образ особенно идёт |
| **works_on_textures** | `straight`, `wavy`, `curly`, `coily` | на каких натуральных текстурах реалистично смотрится/достижим |
| **suits_hair_density** | `fine` (тонкие/редкие), `medium`, `thick` (густые) | какой густоте льстит образ |

### 2.1. Справочник «форма лица → что идёт» (для разметки и ранжирования)

- **oval** — универсально, идёт почти всё (сбалансированные пропорции).
- **round** — добавлять высоту/вытягивать: длинные слои, боковой пробор, объём сверху, длина ниже подбородка; избегать ширины у щёк и коротких ровных бобов на линии скул.
- **square** — смягчать углы: волны, слои, косая чёлка, мягкие края; избегать ровного среза на линии челюсти.
- **heart** (широкий лоб, узкий подбородок) — добавлять ширину у челюсти: бобы/лобы до подбородка, косая чёлка; избегать тяжёлого объёма сверху.
- **oblong/long** — добавлять ширину, укорачивать визуально: чёлки, ровные срезы, волны; избегать очень длинных гладких прямых без чёлки.
- **diamond** (узкий лоб и подбородок, широкие скулы) — чёлки и длина до подбородка; добавлять ширину у лба и подбородка.

> Форму лица планируем **детектить автоматически по фото**, с возможностью ручной правки в визарде.

## 3. Схема карточки образа (entity `Hairstyle`)

```jsonc
{
  "id": "uuid",
  "slug": "french-bob",
  "name": { "ru": "Французский боб", "es": "...", "de": "..." },
  "description": { "ru": "Короткий боб до подбородка с лёгкой небрежностью", "...": "" },

  // структурные теги (раздел 1)
  "length": "chin",
  "silhouette": "blunt",
  "texture": ["straight", "wavy"],
  "fringe": "blunt",
  "layering": "one-length",
  "parting": "middle",
  "finish": ["natural", "tousled"],
  "volume": "medium",
  "gender_presentation": "feminine",
  "aesthetic": ["classic", "trendy"],
  "era": "modern",
  "occasion": ["everyday", "work"],
  "maintenance": "low",

  // подходящесть (раздел 2) — для ранжирования
  "flatters_face_shapes": ["oval", "heart", "square"],
  "works_on_textures": ["straight", "wavy"],
  "suits_hair_density": ["medium", "thick"],

  // ассеты (см. раздел 3.1)
  "preview_image": "s3://.../french-bob-preview.jpg",        // С лицом/моделью — для выбора в UI
  "reference_image": "s3://.../french-bob-ref-faceless.jpg", // БЕЗ лица — подаётся в генерацию
  "prompt_template": "{base_enhance}, apply hairstyle: classic french bob, chin length, blunt cut, soft middle part, {color_fragment}; keep the person's face, identity and skin unchanged; only change the hair; photorealistic",

  // служебное
  "tags": ["bob", "short", "chic"],
  "is_active": true,
  "sort_order": 10
}
```

## 3.1. Подача в генерацию и борьба с подмешиванием лица

> **Provisional** — рабочая гипотеза по опыту Валерии, подтвердить у неё.

**Подход к генерации: гибрид — референс образа + промпт.** (Валерия получала «супер» результаты именно так.)

**Критическое требование: референс причёски — БЕЗ лица.**
- Проблема *identity bleed*: при наличии лица в референсе модель подмешивает чужие черты к лицу пользователя → результат «не он». Реальная боль.
- Решение: подаём в генерацию изображение **только с волосами, без лица**.

**Две картинки на карточку (не одна):**
- `preview_image` — с лицом/моделью, красивое, для выбора в UI.
- `reference_image` — безликое, для подачи в модель.

**Как получать так, чтобы они совпадали:**
- Генерим красивое превью с моделью → для референса **программно маскируем/убираем область лица** (face-detect → маска/кроп). Волосы идентичны превью, лица для подмешивания нет.
- Альтернатива (хуже): генерить на «безликом манекене» — но манекенные волосы выглядят как парик и это может перетечь в результат. Маскирование живого портрета натуральнее.

**Защита от дрейфа личности (в промпте):**
- Явная инструкция сохранять лицо: `keep the person's face, identity and skin unchanged; only change the hair`.
- Безликий референс + эта инструкция = двойная защита.

**Открытые вопросы (уточнить у Валерии / тестом):**
- [ ] Подтвердить: референс + промпт vs только промпт.
- [ ] Достаточно ли маскирования лица, или нужен чистый «hair-only» вырез (сегментация волос).
- [ ] Лучший ракурс безликого референса (анфас с маской / ¾ / затылок) под разные типы причёсок.

## 4. Цвет волос — отдельный слой (entity `ColorOption`)

```jsonc
{
  "id": "uuid",
  "slug": "honey-balayage",
  "name": { "ru": "Медовое балаяж", "...": "" },
  "family": "natural",            // natural | fashion
  "level": 7,                      // 1 (чёрный) … 10 (платина); null для fashion
  "technique": "balayage",         // solid | balayage | highlights | ombre | money-piece | roots
  "hex_swatch": "#C19A6B",
  "prompt_fragment": "warm honey balayage, soft face-framing highlights"
}
```

Базовые семейства:
- **Натуральные уровни:** чёрный, тёмно-коричневый, коричневый, светло-коричневый, тёмный блонд, блонд, платина, рыжий/медный, седой/серебро.
- **Техники:** solid, балаяж, мелирование, омбре, money-piece, тёмные корни.
- **Fashion (опц., позже):** пастель, яркие (розовый, синий…).

## 5. Логика фильтрации (визард → каталог)

Визард собирает два набора (детали — `PRODUCT.md`):
- **Текущие волосы:** длина, текстура, густота, текущий цвет, форма лица (детект).
- **Предпочтения:** желаемая длина, вайб/повод, готовность к уходу, желаемый цвет, гендер-подача, «смело / безопасно».

**Жёсткие фильтры (отсечение):**
- `gender_presentation` ≠ выбранному (если выбран).
- `length` вне желаемого диапазона (если пользователь явно ограничил).
- `maintenance` выше потолка готовности к уходу (если задан).

**Мягкое ранжирование (score, не отсечение):**
- + совпадение `flatters_face_shapes` с детектированной формой лица.
- + совпадение `works_on_textures` с текстурой пользователя (достижимость без радикальной смены).
- + совпадение `suits_hair_density` с густотой.
- + совпадение `aesthetic` / `occasion` с предпочтениями.
- небольшой буст по `popularity`/`sort_order`.

Показываем топ-N по score (например 6–12), с возможностью «показать ещё». Цвет выбирается отдельно и не участвует в фильтрации образов.
